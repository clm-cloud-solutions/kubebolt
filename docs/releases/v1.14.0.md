# KubeBolt 1.14.0 ‚Äî harden what's shipped, round out visibility

1.14 is a deliberately light release. A code audit at the start of the
cycle found that **Kobi already proposes, executes, and verifies live
remediations** ‚Äî that capability landed quietly across the 1.12‚Üí1.13
line. So instead of building new agency, 1.14 **hardens the action
surface that already ships** (governance + a durable audit trail) and
**rounds out visibility** into the corners of a cluster KubeBolt didn't
yet show: Helm releases, five standard CRDs, and more incident-detection
insights. The scope was kept tight on purpose to free capacity for the
KubeBolt SaaS v1 launch.

Backend ships as **KubeBolt 1.14.0**. The agent is **unchanged this
cycle** ‚Äî it stays at **kubebolt-agent 1.1.0** (no agent image or chart
bump). See the [compatibility matrix](#compatibility-matrix) for
supported pairings.

> **No breaking changes.** 1.14 is a clean `helm upgrade` from 1.13.x.
> New BoltDB buckets (`insights`, `kobi_actions`) are created
> automatically on first boot; the chart adds a few additive RBAC grants
> (see [Upgrade](#upgrade)). Nothing requires a values change.

## What ships

### Insights persistence ‚Äî stable identity + history (Sprint 0)

Insights used to be ephemeral and anonymous: in-memory only, lost on
restart, with a fresh UUID every evaluation tick and no tenant/cluster
identity ‚Äî yet they're the origin for notifications, the insight‚ÜíKobi
trigger, and Autopilot.

- **Stable identity** ‚Äî a deterministic `fingerprint =
  sha256(tenantID|clusterID|ruleID|resource)` plus a per-episode
  occurrence ULID for history.
- **`InsightStore` + BoltDB impl** ‚Äî a new `insights` bucket keyed
  `tenantID/clusterID/fingerprint`, JSON records, hourly prune ticker,
  hot-reloadable retention via `KUBEBOLT_INSIGHTS_RETENTION_HORIZON`.
  `tenantID="default"` in OSS; SaaS swaps in a Postgres impl with zero
  rekey migration.
- **Restart-safe** ‚Äî active + recently-resolved insights persist with
  `firstSeen`/`lastSeen`/`resolvedAt`. This also kills the
  re-notify-spam bug where a restart re-fired notifications for insights
  that were already open.
- **History API** ‚Äî `GET /insights?history=true&status=&since=&until=`.
  Default behavior unchanged. History is **API-only in 1.14** (no UI
  surface yet); it's the foundation for future UI / SaaS / Autopilot.

### Kobi action governance & audit (Sprint 1)

The propose‚Üíexecute‚Üíverify‚Üíconfirm loop already ships (8 mutating verbs,
RBAC-scoped, with live rollout verification and typed confirmation on
destructive ops). 1.14 puts governance and a durable record around it:

- **Persistent audit trail** ‚Äî a new BoltDB `kobi_actions` bucket. Every
  executed proposal writes a structured record: timestamp, user, role,
  cluster, action, target, params, before/after snapshot, result,
  duration, and source (`copilot_proposal` vs `ui`).
- **Admin audit history UI** ‚Äî a chronological, filterable list under
  `/admin` with before/after diff.
- **Governance toggle** ‚Äî an admin setting with two knobs: a master
  switch (off ‚Üí Kobi reverts to pure read-only advisory) and a
  destructive-ops sub-switch (off ‚Üí delete and scale-to-0 are withheld
  from the tool list and rejected server-side). **Default ON.**
- **Consolidated action provider** ‚Äî the scattered verb logic is unified
  behind a single Go interface (`Describe`/`DryRun`/`Apply`) in
  `internal/copilot/actions/`, with the verb whitelist, audit hook, and
  toggle enforcement single-sourced.
- **New verb ‚Äî `attach-debug-container`** ‚Äî Kobi can offer an ephemeral
  debug container during triage.

### Helm releases ‚Äî read-only first-class (Sprint 4)

- **No SDK import.** Release state is read by decoding Helm's own release
  Secrets directly (base64 ‚Üí gzip ‚Üí JSON via the existing secret lister),
  avoiding the `helm.sh/helm/v3` bloat and Trivy-gate risk that the write
  path would carry. New component `internal/helm/releases.go`, synced on a
  30s ticker.
- **REST** ‚Äî `GET /helm/releases` and
  `GET /helm/releases/{namespace}/{name}` (values + manifest + history).
- **Applications** ‚Äî a new sidebar nav item with a distinct-releases
  counter. List view (chart / version / status / last deployed / age) and
  a detail view with Values (read-only YAML), a per-resource **manifest
  explorer** (resizable master-detail), History, and Related.
- **Insights** ‚Äî `helm-release-failed` (last install/upgrade ended
  `failed`) and `helm-release-hook-pending` (pre/post hook stuck >5m).

Helm **write** verbs (upgrade / rollback / uninstall) and the App Center
are deliberately out of scope (deferred post-1.14).

### Remote-friendly port-forward (Sprint 2)

Port-forward now works against **remote backends** (in-cluster /
agent-proxy deployments), not just when the backend and browser are the
same machine.

- HTTP preview through a `/pf/{id}/` reverse proxy plus a WS raw-stream
  bridge, carried over the agent's existing gRPC tunnel
  (`TunnelConn` / multiplexed streams).
- Upstream redirects are rewritten by hostname (ignoring the port) so a
  backend behind nginx's default `absolute_redirect` no longer leaks
  `127.0.0.1` to the browser.
- **Known limitation:** apps that hardcode absolute root-path assets
  (`/static`, `/api`, ‚Ä¶) without sub-path awareness ‚Äî Grafana, Argo CD,
  etc. ‚Äî load their shell but not their assets through `/pf/{id}/`. The
  proxy rewrites the `Location` header, not the response body. Use the
  app's own sub-path config; full raw transport is deferred to 1.15. The
  UI says so inline.

### Standard CRD rich views (Sprint 3)

Five well-known types become first-class (list + detail + insights):

- **PodDisruptionBudget** ‚Äî completes the Evict story (Evict blocked ‚Üí
  PDB detail with currentHealthy / allowed / selector). Insight: PDB
  selector matches zero pods.
- **HorizontalPodAutoscaler** ‚Äî current/desired replicas, scaling
  timeline, target metric ladder.
- **VerticalPodAutoscaler** recommendation (if VPA installed) ‚Äî
  Recommended vs Current, read-only.
- **cert-manager Certificate** ‚Äî issuer, expiration, renewal countdown.
- **ArgoCD Application** ‚Äî sync status, last reconciliation, diff summary.

**ServiceAccount** also became first-class (fixes the Helm Related link),
and Settings' "Service Accounts" tab was renamed **API Tokens** with CRD
counters.

### New incident-detection insight rules (Tier-1)

Four rules added to the OSS insight engine (which Autopilot consumes as
its trigger source):

- **`progress-deadline-exceeded`** (critical) ‚Äî the canonical
  stuck-rollout signal (`Progressing=False / ProgressDeadlineExceeded`).
- **`missing-config-dependency`** (critical) ‚Äî container
  `CreateContainerConfigError` (a referenced ConfigMap/Secret is missing);
  reads live container state, so it clears the moment the dependency
  exists.
- **`readiness-probe-failing`** (warning) ‚Äî Running but `Ready=False`
  past a 2-min grace ("up but not serving").
- **`liveness-probe-failing`** (warning) ‚Äî recurring kubelet `Unhealthy`
  events. Hardened so it does not fire for an already-deleted pod (Events
  outlive their object by ~1h).

### Workload revision YAML diff (History tab)

A **View Diff** action per revision row opens a side-by-side YAML diff
(GitHub-style red/green tints), diffing K8s-native revision history
(ReplicaSets / ControllerRevisions) ‚Äî no snapshot store, so it captures
every template change regardless of who made it. Rollback to either side
reuses the gated flow. Workloads only (Deployments / StatefulSets /
DaemonSets).

### Polish

- Insights list is **paginated** (10/page default + per-page selector;
  no more endless scroll).
- Pod Overview shows the restart sparkline, not just the Containers tab.
- Chart RBAC: `pods/portforward` + `pods/eviction` grants, `create` for
  all New-resource modal kinds, and read for PDBs + the optional 1.14
  CRDs.

### Security

- **API / binaries rebuilt on Go 1.25.11** (released 2026-06-02),
  clearing CVE-2026-27145 / -42504 / -42507 (crypto/x509, mime,
  net/textproto) that Artifact Hub flagged against the 1.13.1 images.
- **Web image** picks up `libxml2 2.13.9-r1` on rebuild, clearing
  CVE-2026-6732 (libxml2 DoS, HIGH).

## Upgrade

### Standard backend upgrade

```bash
helm upgrade kubebolt oci://ghcr.io/clm-cloud-solutions/kubebolt/helm/kubebolt \
  --version 1.14.0 \
  --reuse-values
```

The upgrade adds a few additive RBAC grants to the KubeBolt ServiceAccount
(`pods/portforward`, `pods/eviction`, `create` on the New-resource modal
kinds, read on PDBs + the optional CRDs). No values change is required.

### Agent

**No agent upgrade in 1.14** ‚Äî the agent is unchanged since
`kubebolt-agent 1.1.0`. If you're already on 1.1.0 there is nothing to do.

### Optional: insights retention

History retention defaults to the same horizon as before. To tune it:

```bash
--set extraEnv.KUBEBOLT_INSIGHTS_RETENTION_HORIZON=168h   # e.g. 7 days
```

## What does NOT ship in 1.14.0

- **Helm write actions** (upgrade / rollback / uninstall), the **App
  Center** (install from OCI/HTTP repos + cosign verification), the
  `helm.sh/helm/v3` SDK import, and the two Kobi Helm verbs that depended
  on them. Deferred post-1.14.
- **An insights-history UI.** History is API-only in 1.14
  (`/insights?history=true`); it's the data foundation for a future UI.
- **Body-rewriting port-forward** for apps with absolute root-path assets
  (Grafana / Argo CD without sub-path config). The raw transport (Option
  C) is on the 1.15 roadmap.
- **Arbitrary A-vs-B revision comparison** and revision diff for
  non-workload kinds (K8s doesn't version ConfigMaps/Services/etc.).
