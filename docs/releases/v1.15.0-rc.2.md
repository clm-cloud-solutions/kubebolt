# KubeBolt 1.15.0-rc.2 — dashboard redesign + pipeline re-validation

> **Release candidate.** rc.2 re-cuts 1.15 for release-pipeline validation
> (image build + Trivy gates + chart publish) after rc.1 flagged security
> findings in the bundled metrics image, and folds in the Overview/Capacity
> dashboard redesign that landed since. Run it in staging; the 1.15.0 GA notes
> will supersede this file. See [rc.1](v1.15.0-rc.1.md) for the full 1.15
> feature set (W1 org→team→user foundation, Kobi maturation + read-only MCP,
> Cilium NetworkPolicy support).

## Security — why the re-cut

- **VictoriaMetrics / vmagent `v1.145.0-rc0` → `v1.145.0` (final).** rc.1 pinned
  the VM `rc0` build, whose image flagged HIGH/CRITICAL CVEs at the preflight
  Trivy gate. VictoriaMetrics has since shipped the **stable v1.145.0** on the
  patched Go runtime; rc.2 moves the pin to it across helm values +
  docker-compose + the agent chart's vmagent (in lockstep) and re-validates that
  the findings are cleared.

## Dashboard redesign (#97)

The Overview and Capacity surfaces were reworked so the whole dashboard reads as
one visual family and so "what's wrong" is drawn, not implied.

- **Overview KPIs as ring gauges.** New shared `DonutGauge` (SVG ring, extracted
  from the Monitor-tab donut) + `LegendRow` (dot · label · value, optional deep
  link) primitives. Health / Nodes / Pods cards become 116px gauges with the
  value centered + breakdown legend rows — same grammar as the usage cards.
- **The "Degraded" bucket is finally visible.** The Pods card surfaces the
  previously-invisible Warning bucket (Pending / CrashLoopBackOff / partially
  ready) as a *Degraded* legend row. Backed by a new **`degraded` pseudo-status**
  in the API's `GetResources` filter — anything neither healthy (Running
  fully-ready / Succeeded) nor terminal-failed, mirroring the overview's Warning
  bucket, so the KPI links to a list that matches its count (including Running
  pods with containers not ready, which no exact status match can capture).
- **Filtered deep links.** `ResourceListPage` now reads `?status=` from the URL
  (chip + clear + empty state + page reset), and `InsightsList` moves its
  severity filter to the URL — so `/insights?severity=warning` and the Degraded
  KPI land pre-filtered.
- **Usage cards** collapse three stacked bars into a graded `Used` donut + one
  bullet bar (Requests fill, Limits marker) with a breakdown hover tooltip.
- **Workload Health** gains a *Needs attention* section listing degraded
  workloads with links to their detail pages; **Namespace tiles** segment the
  bottom bar ready/not-ready so a broken pod is rendered, not implied.

## Capacity trends — request / limit / capacity overlays (#97)

- The Capacity CPU-Usage and Memory-Working-Set charts gain cluster-wide
  **request + limit reference lines** plus a **capacity (allocatable)** line.
  `buildCpuRefs` / `buildMemRefs` / `formatMemoryShort` were extracted into
  `utils/metricRefs.ts` and shared with `ResourceDetailPage` so both surfaces
  render identical overlays.
- `MetricChart` learns `ReferenceLineSpec.defaultHidden` (capacity ships hidden
  — it sits an order of magnitude above usage and would squash the curves; a
  header pill reveals it on demand) and `refsPersistKey` to persist toggle state
  in `localStorage`, keyed by stable `shortLabels` so a moving threshold value
  no longer resets the hidden state.

## `endpoints` dual-key resolution fix (#96)

KubeBolt surfaces EndpointSlices under the `endpoints` type, but slices are named
`<service>-<hash>` (e.g. `shop-db-vvgwz`), not the service name — and read paths
diverged: the web UI navigates by the slice name, while **Kobi / Autopilot
navigate by the fronting Service name**. Each path handled only one key, so the
other 404'd — and a 404 was misread downstream as "the endpoint registration was
deleted," steering RCA toward *recreate the Endpoints / redeploy the workload*
when the backing external host was simply down (exactly the
selectorless-external-DB trap the 1.15 Kobi work hardens against).

Every `endpoints` read path now accepts **both** keys — `GetResourceYAML`,
`GetResourceDetail`, and describe (REST + copilot executor, via a new
`Connector.ResolveEndpointsName`) resolve either the slice name or the
`kubernetes.io/service-name` label to the real slice.
