# KubeBolt 1.17.0 — metrics-only clusters, flexible UI hosting & UX polish

1.17.0 is a feature release focused on **connecting clusters that only ship metrics**,
**hosting the web UI separately from the API**, and a round of interface polish for
small screens and everyday navigation. It's single-tenant and drop-in within the 1.x
line — **no schema change**.

The node agent moves to **kubebolt-agent 1.1.6** — a reliability-only bump on the
**same metric/label schema**, a drop-in within the **1.1.x** line.

---

## Metrics-only cluster tier

Clusters can now connect in a **metrics-only** mode — VictoriaMetrics ingest (via the
agent's promread / DaemonSet path) **without** the live agent-proxy channel — for
setups that only need dashboards:

- **Mode detection & connect-path hardening** so the manager wires only what the mode
  supports.
- **Graceful UI degradation** — connection-type badges, a metrics-only Capacity gate, a
  stale reader/operator badge, and **Recent Deploys derived from KSM** (no live API).
- **Freshness-based liveness** — based on VictoriaMetrics sample freshness rather than
  the absent channel.

## Configurable API origin

`VITE_API_URL` sets the API origin at build time, so the SPA can be hosted separately
(a static/edge host) and talk to a **remote** API. Empty keeps the unchanged
same-origin behaviour (nginx proxies `/api`). REST, the **live-updates WebSocket**, the
**pod-exec terminal**, and **Copilot** all honour it (https→wss for the sockets).
`KUBEBOLT_CORS_ORIGINS` (comma-separated, wildcards allowed) overrides the CORS
allowlist for cross-origin browser calls — `go-chi/cors` reflects the matched origin, so
it's credentials-safe.

## UI/UX

- **Collapsible left-nav groups**, persisted to localStorage (first visit opens
  Workloads only; Pinned always visible).
- **Feedback link** in the sidebar footer that opens the feedback form with the
  signed-in email prefilled.
- **Responsive handling** — a full-screen "best on a larger screen" notice below 768px,
  a dismissible banner on tablets (768–1024px), and a **topbar that degrades to icons**
  near the 1024px minimum (Search / New / Dashboard / Cluster Map collapse; the cluster
  name truncates). The console is desktop-first, so this sets expectations instead of
  breaking quietly.
- **Product positioning unified** to "AI operations platform for Kubernetes" across the
  UI and Kobi prompts.

## Fixed

- **Agent-proxy reliability** — gRPC keepalive + force-reconnect for stuck agents
  (bounded, configurable), **JSON watch frames** (no more protobuf-decoded-as-empty-JSON
  relist storm), and over-fetch caching for helm/overview/gateway calls.
- **Insights lifecycle** — resolve lingering/flapping insights and show the real
  first-seen age.
- **Investigation UX** — logs default to **off** (opt-in "Live" tail); the overview no
  longer blocks on a live ServerVersion call; the refresh cookie sets `Secure` on HTTPS.
- **Agent (1.1.6)** — UTF-8 sanitize on the AgentChannel, stats interval aligned to the
  30s dedup window (was 15s → 6× over-scrape), dummy interfaces filtered in both network
  collectors, single-scrape KSM, and an EOF-reconnect + buffer-stall watchdog in the
  shipper. See [`packages/agent/CHANGELOG.md`](../../packages/agent/CHANGELOG.md).

## Upgrading

Drop-in within 1.x — no schema change. If you serve the UI same-origin (the default),
nothing changes; `VITE_API_URL` and `KUBEBOLT_CORS_ORIGINS` are opt-in.

```bash
helm upgrade kubebolt-agent \
  oci://ghcr.io/clm-cloud-solutions/kubebolt/helm/kubebolt-agent --version 1.1.6
```
