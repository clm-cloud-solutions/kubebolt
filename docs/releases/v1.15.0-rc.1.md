# KubeBolt 1.15.0-rc.1 — multi-tenant foundation + Kobi grows up

> **Release candidate.** 1.15.0-rc.1 is cut for release-pipeline validation
> (image build + Trivy gates + chart publish) ahead of the 1.15.0 GA. Run it in
> staging; the GA notes will supersede this file.

1.15 lays the **foundation for KubeBolt SaaS / Enterprise** while staying a
clean, behavior-compatible upgrade for OSS. The headline is **W1** — the
store-interface seams and the now-operational **organization → team → user**
hierarchy — alongside a big round of **Kobi** maturation (a read-only MCP
server + a dozen copilot UX fixes) and **Cilium NetworkPolicy** support.

Backend ships as **KubeBolt 1.15.0-rc.1**. The agent is **unchanged this
cycle** — it stays at **kubebolt-agent 1.1.0** (no agent image or chart bump).

> **No breaking changes.** Clean `helm upgrade` from 1.14.x. OSS behavior is
> unchanged except that the org → team → user hierarchy is now *materialized*
> (every user becomes a real member of the auto-seeded default team on first
> boot — idempotent, additive) and the single-org/single-team boundary is now
> enforced server-side. New BoltDB buckets are created automatically. Bundled
> VictoriaMetrics + vmagent are on **v1.145.0-rc0** (carries the CVE-2026-42504
> fix already shipped in 1.14.1).

## What ships

### W1 — multi-tenant store seams + operational org → team → user hierarchy

The groundwork for SaaS/EE, landed entirely behind seams so OSS is unaffected:

- **Store-interface seams** (interface in OSS + BoltDB default; EE swaps a
  Postgres impl): `UserStore`, `TenantStore` (organization), `TeamStore`
  (+ `Team` / `Membership` / `EffectiveRole`), `ClusterStore`,
  `RefreshTokenStore`, `APITokenStorer`, and a new `UsageStore` metering seam
  (no-op in OSS).
- **IngestTokenStore refactor** — the long-lived `kb_` ingest tokens moved out
  of the inlined `Tenant.IngestTokens` field into their own store, with a
  one-time idempotent boot migration (clean cutover, no duplication).
- **Operational hierarchy (OSS = single org + single team):** boot ensures the
  default team and backfills every user into it; `CreateUser` / `DeleteUser`
  keep membership in sync; `/auth/me` returns the user's org + team (effective
  role); new read-only `/teams`, `/teams/{id}`, `/teams/{id}/members`.
- **OSS↔EE boundary enforced server-side** via the `MultiTenantEnabled` seam:
  creating a second organization or team returns **409 `requires_ee`** so the
  UI can surface an upgrade CTA; EE flips the seam to unlock it.

### Administration UI — regrouped into domain hubs

The admin area moved from a flat 10-item list + monolithic Settings page into
**domain hubs** (tabbed shells, URL-driven `?tab=`): **Access** (Users · Teams ·
Authentication), **Agents & Ingest** (Agent Tokens · Activity · Integrations ·
Configuration), **AI (Kobi)** (Configuration · Usage), **System** (General ·
Notifications), plus standalone **API Tokens**. The user menu now shows the
operator's org + team, and there's a real **Teams** page (members + effective
role).

### Kobi — a read-only MCP server + copilot UX maturation

- **Read-only Kobi MCP server** (HTTP + stdio transports) — exposes Kobi's
  read tooling to external MCP clients, multi-tenant aware, service-token gated.
- **Copilot UX:** Kobi Usage breakdown (per-user / per-trigger + reliability
  strip), configurable tool-call rounds with a graceful close + step-limit
  notice and a **Continue** control, configurable action-progress timeout with
  auto-root-cause on stall, in-flight turn cancel + streaming latency feedback,
  automatic dry-run preview on action proposals, a `debug_pod` command, and a
  fix that stops resumed/historical proposals from re-investigating a stall.

### Cilium NetworkPolicies — first-class resources

`CiliumNetworkPolicy` and `CiliumClusterwideNetworkPolicy` are now first-class
resources (list + detail), with a policies UI (grouped / complete views + a
"New" picker). Also fixes a create-from-manifest GVR-map sync gap.

### Incident-simulation lab (docs)

A documented Kobi incident-simulation lab — a set of scenarios plus a
three-tier cascade app — for exercising detection + remediation end to end.

## Upgrade

`helm upgrade` from 1.14.x is clean — no values change required. The default
team + membership backfill run automatically on first boot and are idempotent.
