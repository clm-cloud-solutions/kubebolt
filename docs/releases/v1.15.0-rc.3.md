# KubeBolt 1.15.0-rc.3 — bundled metrics on the scratch image

> **Release candidate.** rc.3 supersedes [rc.2](v1.15.0-rc.2.md) with one change:
> the bundled VictoriaMetrics / vmagent now run the **`-scratch`** image variant.
> Run it in staging; the 1.15.0 GA notes will supersede this file. See
> [rc.1](v1.15.0-rc.1.md) and [rc.2](v1.15.0-rc.2.md) for the full feature set
> (W1 org→team→user foundation, Kobi maturation + read-only MCP, Cilium
> NetworkPolicy support, the Overview/Capacity dashboard redesign, and the
> `endpoints` dual-key fix).

## Why rc.3

- **VictoriaMetrics + vmagent `v1.145.0` → `v1.145.0-scratch`.** The default
  (Alpine-based) v1.145.0 image carries **CVE-2026-45447** — a HIGH/Critical
  OpenSSL `PKCS7_verify()` use-after-free in `libcrypto3` / `libssl3`, disclosed
  2026-06-09 and fixed in OpenSSL 3.5.7 / alpine 3.5.7-r0. The VM image predates
  that fix, and no patched VM image exists yet.

  KubeBolt does not exercise the vulnerable path (the bundled TSDB is
  cluster-internal and never does PKCS#7/S-MIME verification), but rather than
  ship — or publicly publish to Artifact Hub — an image flagged for a
  high-profile CVE, rc.3 switches to VictoriaMetrics' **`-scratch`** variant:
  the application binary only, **no OS packages, no OpenSSL**. Both images scan
  **clean** (0 CRITICAL/HIGH, Trivy `--ignore-unfixed`), on our release gate and
  on the Artifact Hub security report. The scratch variant is VictoriaMetrics'
  production-recommended flavor; KubeBolt runs it as a drop-in (same entrypoint
  + flags), verified against the bundled stack.

Agent unchanged — stays **kubebolt-agent 1.1.0**. No breaking changes — clean
`helm upgrade` from 1.14.x.
