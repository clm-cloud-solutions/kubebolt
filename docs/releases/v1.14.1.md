# KubeBolt 1.14.1 — security patch (bundled VictoriaMetrics)

A patch over [1.14.0](v1.14.0.md) with a single change: the bundled
**VictoriaMetrics** and **vmagent** images are bumped to **v1.145.0-rc0** to
clear a Go-stdlib vulnerability disclosed after the 1.14.0 pin was set. No
application code, API, agent, or chart-template changes — a clean
`helm upgrade` from 1.14.0.

## Security

- **VictoriaMetrics / vmagent `v1.143.0` → `v1.145.0-rc0`** — clears
  **CVE-2026-42504** (HIGH, `net/textproto` / MIME-header decoding DoS in the Go
  standard library). The vulnerability was rated UNKNOWN when 1.14.0 was pinned
  and has since been reclassified to HIGH with an upstream fix (Go 1.25.11 /
  1.26.4); v1.145.0-rc0 is the first VictoriaMetrics build on the patched Go
  runtime. The KubeBolt **api** and **agent** binaries were already on Go
  1.25.11 as of 1.14.0, so only the bundled third-party image needed the bump.
  Both pins move in lockstep (helm values + docker-compose + the agent chart's
  vmagent) per the single-source-of-truth map; the release preflight Trivy gate
  now passes.

## Upgrade

```bash
helm upgrade kubebolt oci://ghcr.io/clm-cloud-solutions/kubebolt/helm/kubebolt --version 1.14.1
```

No configuration changes. The agent is unchanged — it stays at
**kubebolt-agent 1.1.0**.
